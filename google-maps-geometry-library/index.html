Google Maps Javascript API - Geometry Library

Overview

The Google Maps Geometry Library comprises of a set number of static methods provided to calculate or determine geometric data on the surface of your Google Map. The methods includes computing values based on the Spherical Namespace, Encoding Namespace and Poly Namespace.

To be able to use this library, please ensure to have the Google Maps Javascript API activated on your API Dashboard post creating your API Key as well as passing the library parameter like the script tag shown below:

<script type="text/javascript" src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&libraries=geometry"></script>

Checkout the examples below and refer to the example codes attached to each of them to get started. To get a better in-depth understanding of how this works, please visit Google Maps Javasript API - Geometry Library for more information.

Google Maps Geometry - Compute Length

To start off with the Geometry Library, we'll calcualte the distance between two points that have been preset on a map just like the example. To get the distance covered between atleast 2 points, use the google.maps.geometry.spherical.computeLength(pathMVCArray) method with its argument being either a Polyline Path Array or an MVC Array containing Lat Lng Objects created with new google.maps.LatLng() class.

The Compute Length method is suitable for calculating distance being 2 or more Latitude-Longitude points on a map, unlike a similar method computeDistanceBetween() for only two points.

In this example, you are given the distance between 2 points on a map. Interact with the map by editing/adding an additional Lat-Lng Coordinate by clicking at the editable links on the Polyline Path. Once set, you would be updated with the total distance between all new points that have been set and also given any new Latitude-Longitude Coordinates that have been set as well.

        Google Maps Geometry - Encoding and Decoding

        While we may be used to parsing Latitude and Longitude details from an array especially when handling multiple way points on our maps, Google Maps Geometry Library has an Encoding (google.maps.geometry.encoding.encodePath(yourPathArray)) method and a Decoding (google.maps.geometry.encoding.decodePath(encodedStringValue)) which can also be used instead. You can choose to encode a path array at one point and then when you're ready to deploy the required coordinates, you could process the encoded string value through the decode method, which helps decompressing it and provides you a usuable array.

        In our example, interact with it by clicking on the map, you would notice that the results area below provides the output at every stage of clicking on the map with an Encoded path array string value. Open your browser console and decode at every step once encoded and you should see the decoded array version of the output.

          Google Maps Geometry - Contains Location

          The Contains Location() Geometry Method is used to find out if a specific location on a map, is within an area that is covered by an overlay like a polygon. The return type value for this method is Boolean. In this example, there is a specfic area that is covered on the map (but not visibly placed on it) with a Polygon. If you were to click within the specified covered area, the containsLocation() method is fired and an animated bouncing marker would be placed on it.

          Please note that this method requires two arguments to be passed into it, one being the event.latLng object and another is the actual overlay shape(new google.maps.Polygon() for this example).

          Google Maps Geometry - Is Location On Edge

          The isLocationOnEdge() method is used to determine if a specific Latitude-Longitude Coordinate is falling at the edge or near the edge of a kind of overlay, like a polygon. This particular method accepts 3 arguments, the event.latLng Object, the overlay object which would be specified (for example, new google.maps.Polygon()) and a tolerance number value. The return value of this method is a Boolean data type.

          In this example, interact with the map by clicking on it, if the click event happens to be close to the edge of the polygon area (specified also with a tolerance value), then a bouncing marker would appear on the map.
