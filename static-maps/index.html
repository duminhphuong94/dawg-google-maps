Google Static Maps API

An embed version for placing google maps onto your web-pages without having to code in Javascript. Please ensure to enabled the Google Static Maps API at https://console.developers.google.com/apis post creating your API key.

Image Placement

Overview

A Google Static Map is created in response to an HTTP request via a URL, the type of file could be either GIF, PNG or JPEG. The google map image is embedded within an <img> tag A google static maps api url is of the following form: https://maps.googleapis.com/maps/api/staticmap?parameters.

Please refer to Google Static Maps Documentation for more information.

Static Maps: Required Parameters

In the example on the showing Moscow, parameters being used to generate these would be Center, Zoom, Size and the respective API Key. The url request to generate the image would be https://maps.googleapis.com/maps/api/staticmap?key=YOUR_API_KEY&center=kremlin+russia&zoom=15&size=1920x450.

Static Maps: Scale Parameter

Applied in default value of 1, other accepted values for this parameters are 2> and 4 [Available for Google Maps APIs Premium Plan Customers]. This parameter is responsible for developing higher resolution images.

The url request for the image generated would be https://maps.googleapis.com/maps/api/staticmap?key=YOUR_API_KEY&center=grand+canyon+usa&zoom=12&size=1920x450&scale=2

Static Maps: Format, Maptype and Lanugage

Available formats for Static Maps are JPEG, GIF and PNG types. Maptypes are available in roadmap, satellite, hybrid and terrain. Language support available only for some supported country tiles.

The url request for the image generated would be https://maps.googleapis.com/maps/api/staticmap?key=YOUR_API_KEY&center=new+delhi+india&zoom=12&size=1920x450&scale=2&format=JPEG&maptype=terrain&language=hindi
