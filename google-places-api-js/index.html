Google Javascript Places API

All the way from basic map creation to some advanced topics covered on Google Maps Javscript API

Overview

As mentioned by google, "The Places service is a self-contained library, separate from the main Maps JavaScript API code". Our HTML document first needs to be initiated with the respective URL, <script type="text/javascript" src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&libraries=places"></script>.

The API example(s) shown below are handled on the client-side. Ensure to activate the Google Maps Javascript API and Google Places API Web Service from your API Dashboard.

Please refer to Google Places API Documentation for more information.

Nearby Search Request: Let's Start - Location, Radius and Type

While this may not be mandatory from a learning perspective, but the results made with the help of this API is best understood by generating a map with Javascript. Not sure how? Click Here to get started.

We'll begin by selecting a location or latitude-longitude cooridates. Here, the location is Tokyo, set with lat-long coordinates, which is then initiated into a map image with Javascript. To create a Places Search, we'll need to create a Service Object with new google.maps.places.PlacesService(map), followed by service.nearbySearch(request, callback).

The request argument is an object and callback refers to a function that would help generate the results of your call. You could then either console.log to view the results on your browser console or create markers to place them on the map itself.

Nearby Search Request: Rank By

The rankBy field has a default value of google.maps.places.RankBy.PROMINENCE which is resposible for sorting out results based on a certain level of Prominence. The location prominence is based on factors like Google's Index, Global Popularity etc.

In this example, the rankBy field is given the value of google.maps.places.RankBy.DISTANCE, providing results in an ascending order by their distance from the specified location field. Please note that you cannot specify the bounds or the radius fields for this type of field result.

Places Search Results Object - Handling Pagination of Object Results

Search results at times can exceed beyond the usual one set of results that Google Maps Places API gives us. In order to be able to access search results beyond this initial result set, a pagination argument needs to be added to the callback function from service.nearbySearch or service.textSearch.

If there are any additional pages of details which are available from this pagination argument, it would be based on the Boolean value of pagination.hasNextPage, further results can be invoked with pagination.nextPage();.
